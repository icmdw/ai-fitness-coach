## 1. 项目概述

### 1.1 项目背景

随着健身文化的普及，越来越多的人开始进行力量训练。然而，大多数健身爱好者缺乏系统的训练记录和专业的指导，导致训练效果不佳或受伤风险增加。本项目旨在开发一个 AI 驱动的健身管理工具，帮助用户科学记录训练数据、获得个性化建议。

### 1.2 项目目标

- 提供便捷的力量训练记录工具
- 实现多维度的数据可视化分析
- 通过 AI 提供个性化的训练建议
- 帮助用户持续进步，避免平台期

### 1.3 目标用户

- 健身爱好者（特别是力量训练）
- 健身教练（用于管理学员）
- 健身房会员

## 3. 数据模型

### 3.1 核心实体

```
User (用户)
├── id: 用户 ID
├── email: 邮箱
├── created_at: 注册时间
└── profile: 个人档案

UserProfile (个人档案)
├── user_id: 关联用户
├── height: 身高 (cm)
├── weight: 当前体重 (kg)
├── body_fat: 体脂率 (%)
├── goal: 健身目标 (增肌/减脂/维持)
└── updated_at: 更新时间

BodyMeasurement (身体数据记录)
├── user_id: 关联用户
├── recorded_at: 记录时间
├── weight: 体重 (kg)
├── body_fat: 体脂率 (%)
├── chest: 胸围 (cm)
├── waist: 腰围 (cm)
├── hip: 臀围 (cm)
├── arm: 臂围 (cm)
└── thigh: 腿围 (cm)

Exercise (训练动作)
├── id: 动作 ID
├── name: 动作名称
├── category: 动作类别 (胸/背/腿/肩/臂/核心)
├── equipment: 所需器械
├── description: 动作描述
└── instructions: 动作要领

TrainingSession (训练记录)
├── user_id: 关联用户
├── session_date: 训练日期
├── duration: 训练时长 (分钟)
├── notes: 训练备注
└── created_at: 创建时间

TrainingSet (训练组记录)
├── session_id: 关联训练记录
├── exercise_id: 关联动作
├── set_number: 第几组
├── reps: 次数
├── weight: 重量 (kg)
├── rpe: 自觉用力程度 (1-10)
└── rest_time: 组间休息 (秒)
```

### 3.2 实体关系图

```
User (1) ── (1) UserProfile
User (1) ── (N) BodyMeasurement
User (1) ── (N) TrainingSession
TrainingSession (1) ── (N) TrainingSet
Exercise (1) ── (N) TrainingSet
```

## 4. API 设计概述

### 4.1 RESTful API 端点

| 端点 | 方法 | 描述 |
|------|------|------|
| `/api/users` | POST | 用户注册 |
| `/api/users/login` | POST | 用户登录 |
| `/api/users/profile` | GET/PUT | 获取/更新个人资料 |
| `/api/body-measurements` | GET/POST | 获取/记录身体数据 |
| `/api/exercises` | GET | 获取动作列表 |
| `/api/training-sessions` | GET/POST | 获取/创建训练记录 |
| `/api/training-sets` | POST | 添加训练组 |
| `/api/stats/progress` | GET | 获取进步曲线数据 |
| `/api/stats/training-volume` | GET | 获取训练量统计 |
| `/api/ai/analyze` | POST | AI 训练状态分析 |
| `/api/ai/suggest` | POST | 获取训练建议 |

### 4.2 响应格式

```json
{
  "success": true,
  "data": { ... },
  "message": "操作成功"
}
```

错误响应：
```json
{
  "success": false,
  "error": {
    "code": "INVALID_INPUT",
    "message": "具体的错误信息"
  }
}
```

## 5. 非功能需求

### 5.1 性能要求

- 页面加载时间 < 2 秒
- API 响应时间 < 500ms
- 支持 1000+ 并发用户

### 5.2 安全要求

- 用户密码加密存储 (bcrypt)
- JWT Token 认证
- HTTPS 传输
- 防止 SQL 注入、XSS 攻击

### 5.3 可用性要求

- 系统可用性 > 99%
- 数据自动备份（每日）
- 支持数据导出

### 5.4 兼容性要求

- 浏览器：Chrome、Firefox、Safari、Edge（最新 2 个版本）
- 移动端：响应式设计，支持主流手机浏览器

## 6. 技术约束

- 前端：React + TypeScript + TailwindCSS
