## 1. 概述

本文档定义 AI 健身力量训练与饮食管理 Web 应用中的数据可视化设计方案，包括图表类型选择、交互设计和数据展示规范。

### 1.1 设计目标

- **直观易懂** — 用户一眼看懂训练进展和身体变化
- **数据驱动决策** — 帮助用户基于数据调整训练计划
- **美观专业** — 符合健身应用的专业形象
- **交互友好** — 支持探索性数据分析

### 1.2 技术选型

- **图表库**: ECharts (功能强大、响应式好、文档完善)
- **备选**: Chart.js (轻量级、简单图表)
- **渲染**: SVG (清晰、可交互)
- **响应式**: 自适应容器大小

---

## 2. 图表类型总览

| 图表类型 | 用途 | 优先级 | 页面位置 |
|----------|------|--------|----------|
| 折线图 | 力量进步曲线、体重变化 | P0 | 统计页、首页 |
| 柱状图 | 训练量对比、肌肉群分布 | P0 | 统计页 |
| 热力图 | 训练频率、时间分布 | P0 | 统计页 |
| 雷达图 | 身体数据、能力评估 | P1 | 统计页、身体数据页 |
| 面积图 | 累计训练量、热量摄入 | P1 | 统计页 |
| 仪表盘 | 目标完成度 | P1 | 首页 |
| 时间线 | 训练历史 | P2 | 训练记录页 |

## 3. 详细图表设计

### 3.1 力量进步曲线图 (折线图)

#### 用途
展示特定动作的最大重量或总训练量随时间的变化趋势。

#### 数据示例
```json
{
  "exercise": "杠铃卧推",
  "data": [
    { "date": "2026-02-01", "weight": 70, "reps": 5, "sets": 3 },
    { "date": "2026-02-08", "weight": 72.5, "reps": 5, "sets": 3 },
    { "date": "2026-02-15", "weight": 75, "reps": 5, "sets": 3 },
    { "date": "2026-02-22", "weight": 77.5, "reps": 5, "sets": 3 },
    { "date": "2026-03-01", "weight": 80, "reps": 5, "sets": 3 }
  ]
}
```

#### 设计规格
```
┌─────────────────────────────────────────┐
│  杠铃卧推 - 最大重量 (kg)         [3M]  │
│                                         │
│  85 ┤                              ●    │
│  80 ┤                          ●        │
│  75 ┤                  ●                │
│  70 ┤          ●                        │
│  65 ┤  ●                                │
│     └─────────────────────────────────  │
│       2/1   2/8   2/15  2/22  3/1       │
│                                         │
│  [卧推] [深蹲] [硬拉] [下拉] (动作切换)  │
└─────────────────────────────────────────┘
```

#### ECharts 配置要点
- **X 轴**: 时间轴（日期），支持缩放
- **Y 轴**: 重量（kg），自动刻度
- **系列**: 单条折线，带数据点
- **提示框**: 显示日期、重量、组数×次数
- **工具栏**: 数据视图、保存图片、缩放
- **交互**: 点击数据点查看详情

#### 配色
- 折线颜色：`#2563EB` (Primary Blue)
- 数据点：实心圆，白色描边
- 网格线：浅灰色 `#E5E7EB`

### 3.2 训练量热力图

#### 用途
展示用户训练频率和强度分布，类似 GitHub 贡献图。

#### 数据示例
```json
{
  "heatmap": [
    { "date": "2026-03-01", "volume": 5000, "duration": 60 },
    { "date": "2026-03-02", "volume": 0, "duration": 0 },
    { "date": "2026-03-03", "volume": 7500, "duration": 75 },
    // ...
  ]
}
```

#### 设计规格
```
┌─────────────────────────────────────────┐
│  训练热力图 (过去 12 个月)                │
│                                         │
│     1 月  2 月  3 月  4 月  ...          │
│  周  ┌─────────────────────────────┐    │
│  1   │ █  ░  █  █  ░  ░  █  ...    │    │
│  2   │ █  █  ░  █  █  ░  █  ...    │    │
│  3   │ ░  █  █  ░  █  █  ░  ...    │    │
│  4   │ █  ░  █  █  ░  █  █  ...    │    │
│      └─────────────────────────────┘    │
│                                         │
│  颜色深浅 = 训练量 (kg)                  │
│  ░ 休息  █ 低  █ 中  █ 高  █ 极高       │
└─────────────────────────────────────────┘
```

#### ECharts 配置要点
- **图表类型**: 自定义热力图或 heatmap 系列
- **X 轴**: 月份
- **Y 轴**: 周数 (1-4 或 1-5)
- **颜色映射**: 5 级渐变（无→低→中→高→极高）
- **提示框**: 显示日期、训练量、训练时长、训练内容

#### 配色方案
```javascript
color: [
  '#F3F4F6',  // 无训练 (灰色)
  '#DBEAFE',  // 低 (浅蓝)
  '#93C5FD',  // 中 (蓝色)
  '#3B82F6',  // 高 (深蓝)
  '#1E40AF'   // 极高 (深蓝)
]
```

### 3.3 身体数据雷达图

#### 用途
展示身体各维度数据的综合评估或对比。

#### 数据示例
```json
{
  "bodyMetrics": {
    "weight": 75.5,      // kg
    "bodyFat": 15.2,     // %
    "muscleMass": 62.3,  // kg
    "bmi": 22.1,
    "waist": 80,         // cm
    "chest": 100,        // cm
    "arm": 35,           // cm
    "thigh": 55          // cm
  }
}
```

#### 设计规格
```
┌─────────────────────────────────────────┐
│  身体数据概览                    [详情] │
│                                         │
│           体重                          │
│             ●                           │
│           ╱   ╲                         │
│     体脂 ●─────● BMI                    │
│         ╱         ╲                     │
│    腰围 ●           ● 胸围              │
│         ╲         ╱                     │
│     臂围 ●─────● 腿围                   │
│           ╲   ╱                         │
│             ●                           │
│           肌肉量                        │
│                                         │
│  当前值 vs 目标值 (双层雷达图)           │
└─────────────────────────────────────────┘
```

#### ECharts 配置要点
- **图表类型**: radar
- **指示器**: 各身体数据维度
- **数据系列**: 
  - 当前值（实线，蓝色填充）
  - 目标值（虚线，灰色）
- **标准化**: 各维度数据归一化到 0-100

#### 配色
- 当前值：`#2563EB` (蓝色，填充透明度 0.3)
- 目标值：`#9CA3AF` (灰色，虚线)

### 3.4 训练量对比柱状图

#### 用途
对比不同时间段、不同肌肉群的训练量。

#### 数据示例
```json
{
  "weeklyVolume": [
    { "week": "W1", "volume": 15000, "sessions": 3 },
    { "week": "W2", "volume": 18000, "sessions": 4 },
    { "week": "W3", "volume": 12000, "sessions": 2 },
    { "week": "W4", "volume": 20000, "sessions": 4 }
  ],
  "muscleGroupVolume": [
    { "group": "胸部", "volume": 8000 },
    { "group": "背部", "volume": 9500 },
    { "group": "腿部", "volume": 12000 },
    { "group": "肩部", "volume": 5000 },
    { "group": "手臂", "volume": 4500 }
  ]
}
```

#### 设计规格
```
┌─────────────────────────────────────────┐
│  周训练量对比 (kg)                       │
│                                         │
│  20k ┤                    █              │
│  15k ┤          █         █              │
│  10k ┤    █     █         █              │
│   5k ┤    █     █     █   █              │
│   0  └─────────────────────────────      │
│         W1    W2    W3    W4             │
│                                         │
│  肌肉群分布                              │
│  ┌─────────────────────────────────┐    │
│  │ 腿部 ████████████████ 12000kg   │    │
│  │ 背部 █████████████ 9500kg       │    │
│  │ 胸部 ███████████ 8000kg         │    │
│  │ 肩部 ███████ 5000kg             │    │
│  │ 手臂 ██████ 4500kg              │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

#### ECharts 配置要点
- **图表类型**: bar (柱状图)
- **方向**: 垂直（时间对比）或 水平（肌肉群对比）
- **堆叠**: 可选（按动作类型堆叠）
- **提示框**: 显示具体数值和详情

#### 配色
- 单一系列：`#2563EB` (蓝色渐变)
- 多系列：分类色板（蓝、绿、紫、橙、红）

### 3.6 目标完成度仪表盘

#### 用途
展示周/月训练目标的完成进度。

#### 数据示例
```json
{
  "weeklyGoal": {
    "target": 4,        // 目标训练次数
    "current": 3,       // 当前完成次数
    "percentage": 75
  },
  "monthlyVolumeGoal": {
    "target": 80000,    // 目标训练量 (kg)
    "current": 65000,
    "percentage": 81
  }
}
```

#### 设计规格
```
┌─────────────────────────────────────────┐
│  本周目标                                │
│                                         │
│        ╭─────────╮                      │
│       ╱    75%   ╲                      │
│      │    3/4    │                      │
│      │   次训练   │                      │
│       ╲         ╱                       │
│        ╰─────────╯                      │
│                                         │
│  [本月训练量：81%]  [本月体重目标：50%]  │
└─────────────────────────────────────────┘
```

#### ECharts 配置要点
- **图表类型**: gauge (仪表盘) 或 progress circle
- **范围**: 0-100%
- **分段颜色**: 
  - 0-50%: 红色
  - 50-80%: 橙色
  - 80-100%: 绿色

## 4. 交互设计规范

### 4.1 通用交互

| 交互 | 行为 | 反馈 |
|------|------|------|
| 悬停 | 显示数据点详情 | 高亮数据点、显示 tooltip |
| 点击 | 选择数据系列 | 高亮选中、更新关联视图 |
| 缩放 | 查看不同时间范围 | 平滑过渡、保持上下文 |
| 拖拽 | 平移视图 | 实时更新、边界限制 |

### 4.2 Tooltip 设计

```
┌─────────────────────────┐
│  2026-03-01 胸部训练     │
│  ─────────────────────  │
│  杠铃卧推               │
│  重量：80 kg            │
│  次数：5 × 3 组         │
│  总量：1,200 kg         │
└─────────────────────────┘
```

- **背景**: 白色，阴影
- **边框**: 圆角 8px
- **文字**: 主要信息加粗
- **位置**: 跟随鼠标，避免遮挡

### 4.3 图例交互

- **点击图例**: 显示/隐藏对应数据系列
- **双击图例**: 只显示该系列
- **拖拽图例**: 重新排序（可选）

### 4.4 数据下钻

```
总览 → 按月 → 按周 → 按天 → 单次训练详情
```

- 支持点击下钻到更细粒度
- Breadcrumb 导航显示当前层级
- 支持返回上一级

## 5. 响应式设计

### 5.1 图表尺寸适配

| 屏幕尺寸 | 图表宽度 | 图表高度 | 备注 |
|----------|----------|----------|------|
| < 640px | 100% | 250px | 单图表 |
| 640-768px | 100% | 300px | 单图表 |
| 768-1024px | 48% | 300px | 双列 |
| > 1024px | 32% | 350px | 三列仪表板 |

### 5.2 移动端优化

- **简化图表**: 减少数据点、隐藏次要元素
- **触摸友好**: 增大交互区域
- **单指操作**: 支持滑动切换时间范围
- **性能**: 限制渲染数据量（最大 100 点）

### 5.3 横屏适配

- 平板横屏时切换为多列布局
- 图表高度增加，展示更多细节

## 6. 数据加载与性能

### 6.1 数据策略

- **懒加载**: 滚动到可视区域再加载图表
- **分页**: 大数据集分页加载
- **缓存**: 已加载数据本地缓存
- **增量更新**: 只更新变化的数据

### 6.2 性能优化

- **数据采样**: 大量数据点时降采样显示
- **Canvas 渲染**: 大数据量使用 Canvas 而非 SVG
- **防抖**: 窗口 resize 时防抖重绘
- **Web Worker**: 复杂计算放到 Worker

### 6.3 加载状态

```
┌─────────────────────────┐
│     加载数据中...       │
│         ⏳              │
│   (或骨架屏占位)        │
└─────────────────────────┘
```

## 8. 技术实现建议

### 8.1 ECharts 封装组件

```tsx
// 基础图表组件
interface ChartProps {
  option: EChartsOption;
  loading?: boolean;
  onReady?: () => void;
  onDataChange?: (data: any) => void;
}

// 使用示例
<LineChart
  option={lineChartOption}
  loading={isLoading}
  height={300}
/>
```

### 8.2 响应式 Hook

```tsx
// 监听容器大小变化
const { width, height, ref } = useChartSize();

// 防抖更新
useDebounce(() => {
  chart.resize();
}, 300, [width, height]);
```

### 8.3 主题配置

```tsx
// 统一主题配置
const chartTheme = {
  colors: ['#2563EB', '#10B981', '#F59E0B', ...],
  backgroundColor: '#FFFFFF',
  textStyle: {
    fontFamily: 'Inter, system-ui',
    color: '#1F2937'
  }
};
```
